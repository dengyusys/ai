# Skill: Save Point

## 技能概述

Save Point 技能帮助用户便捷地记录项目进展并更新记忆系统，创建知识检查点。适用于：
- 任务完成时记录成果
- 长时间对话后总结进展
- 项目状态变化时保存快照
- 定期更新记忆系统

## 核心行为规范

### 1. 激活机制

**显式激活：**
- 用户明确要求："记录进展"、"保存检查点"、"更新记忆"、"save-point"
- 用户提及："我想记录一下"、"更新一下记忆"

**自动检测：**
- 检测到任务完成（用户表示完成某项工作）
- 长时间对话后（超过10轮交互）
- 项目状态明显变化（添加新功能、修复bug、架构调整）
- 用户开始新话题或切换任务时

### 2. 进展收集流程

激活后，通过以下5个结构化问题收集信息：

1. **任务完成情况**："你刚刚完成了什么任务？"
2. **关键成果**："有哪些具体的成果或进展？"
3. **重要决策**："做出了哪些重要的设计或架构决策？"
4. **遇到的问题**："遇到了什么困难或挑战？如何解决的？"
5. **下一步计划**："接下来打算做什么？"

### 3. 记忆文件更新机制

**每日笔记更新：**
- 文件路径：`memory/daily/YYYY-MM-DD.md`
- 更新位置：在"已完成"部分添加新条目
- 格式：`- [x] 任务描述（YYYY-MM-DD）`
  - 具体成果说明（缩进2个空格）

**长期记忆更新：**
- 文件路径：`memory/MEMORY.md`
- 更新位置：在"重要决策记录"部分添加重要决策
- 格式：`- 决策描述（YYYY-MM-DD）`
  - 决策背景和理由（缩进2个空格）

### 4. 检查点创建原则

- **及时性**：进展发生后尽快记录
- **完整性**：包含任务、成果、决策、问题、计划五个维度
- **简洁性**：每个条目不超过3行，重点突出
- **可追溯性**：包含日期标记，便于历史追踪

## 响应特点

### 交互风格
- **简洁直接**：问题清晰，不冗余
- **引导式**：通过结构化问题引导用户思考
- **确认式**：整理信息后让用户确认
- **反馈式**：更新完成后提供简要反馈

### 记录层次
1. **任务级别**：具体完成的工作
2. **成果级别**：可验证的产出
3. **决策级别**：影响项目方向的选择
4. **问题级别**：遇到的挑战和解决方案
5. **计划级别**：下一步行动方向

### 文件处理
- **自动日期检测**：使用当前系统日期
- **文件存在检查**：检查目标文件是否存在
- **模板创建**：文件不存在时创建标准模板
- **格式保持**：保持现有文件格式和结构
- **错误处理**：文件操作失败时提供手动操作指导

## 实现细节

### 文件模板管理

**每日笔记模板：**
```markdown
# 每日笔记 - YYYY-MM-DD

## 今日上下文
- [当前项目状态或上下文]

## 重要决策
1. [重要决策记录]

## 用户偏好记录
- [用户偏好或习惯]

## 已完成
- [x] 任务描述（YYYY-MM-DD）
  - 具体成果说明

## 待办事项
- [ ] 下一步计划
```

### 日期处理逻辑
- 使用系统当前日期，格式：`YYYY-MM-DD`
- 日期作为文件命名和内容标记
- 跨日期时自动创建新文件

### 错误处理策略
1. **文件不存在**：创建模板文件
2. **权限问题**：提示用户手动操作
3. **格式错误**：保持现有格式，仅追加内容
4. **用户拒绝**：跳过记录，不强制

### 智能建议功能
- **进度提醒**：长时间未记录时建议创建检查点
- **成果总结**：帮助用户提炼关键成果
- **计划建议**：基于当前进展建议下一步

## 哲学基础

### 记录的价值
- **知识沉淀**：将临时记忆转化为持久知识
- **进度可视化**：清晰看到项目进展
- **决策追溯**：记录重要决策的背景和理由
- **模式识别**：长期积累后识别工作模式

### 设计原则
1. **最小干扰**：简洁交互，不打断工作流
2. **用户控制**：用户始终可以跳过或拒绝
3. **自动化**：尽可能自动化文件操作
4. **一致性**：保持与现有系统格式一致

### 适用场景
- **个人项目**：跟踪个人学习或项目进展
- **团队协作**：记录团队决策和进展
- **知识管理**：构建个人知识库
- **回顾反思**：为定期回顾提供素材

## 重要提醒

### 隐私和安全
- 仅记录用户明确同意的信息
- 不记录敏感信息（密码、密钥、个人隐私）
- 文件存储在本地，用户完全控制

### 用户控制
- 用户可以随时跳过记录
- 用户可以编辑或删除已记录内容
- 用户可以禁用自动检测功能

### 技能边界
- 不替代专业项目管理工具
- 不强制记录，保持自愿原则
- 不进行复杂分析，仅做简单记录
