# 模板系统说明

## 概述
个人知识管理系统的模板模块，提供标准化的对话、知识和输出格式模板，确保系统的一致性和效率。

## 模板目录结构

### 1. 对话模板 (conversation/)
标准化的问题处理和对话流程：
- **qa-patterns.md** - 问答模式模板（定义类、方法类、比较类、故障排除类）
- **analysis-frames.md** - 分析框架模板（待创建）
- **brainstorming.md** - 头脑风暴模板（待创建）

### 2. 知识格式模板 (knowledge-format/)
标准化的知识记录格式：
- **note-structure.md** - 笔记结构模板（学习笔记、会议记录、项目计划）
- **article-outline.md** - 文章大纲模板（待创建）
- **project-plan.md** - 项目计划模板（已在note-structure.md中包含）

### 3. 输出格式模板 (output-format/)
规范化的输出和报告格式：
- **summary.md** - 总结报告模板（待创建）
- **report.md** - 详细报告模板（待创建）
- **presentation.md** - 演示文稿模板（待创建）

## 模板设计原则

### 1. 一致性原则
- 相同类型的内容使用相同模板
- 模板字段命名统一规范
- 格式和样式保持一致

### 2. 实用性原则
- 模板字段都是必要且有用的
- 避免过度设计和不必要的复杂度
- 模板易于使用和填充

### 3. 扩展性原则
- 模板支持自定义字段
- 模板之间可以组合使用
- 支持模板的继承和覆盖

### 4. 可读性原则
- 模板结构清晰明了
- 字段说明详细易懂
- 示例充分且实用

## 模板使用流程

### 1. 模板选择
根据内容类型选择合适的模板：
- **知识记录** → knowledge-format/
- **问题处理** → conversation/
- **报告输出** → output-format/

### 2. 模板填充
按照模板要求填写内容：
1. 复制模板结构
2. 替换占位符内容
3. 补充必要信息
4. 删除不适用部分

### 3. 模板验证
检查填充后的内容：
- 必填字段是否完整
- 格式是否符合要求
- 内容是否准确无误

### 4. 模板保存
保存使用模板生成的内容：
- 按知识库目录结构存放
- 建立与其他知识的关联
- 记录模板使用情况
