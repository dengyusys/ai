# 个人知识管理系统 - CLAUDE.md

## 系统概述
这是一个个人知识管理系统，通过AI代理（Claude Code）帮助用户组织、检索和扩展个人知识。核心组件：知识库(knowledge)、模板系统(template)、操作日志(log)。

## AI语言风格指南

### 沟通原则
1. **中文优先**：主要使用中文沟通，技术术语保持英文原文
2. **知识导向**：回答基于知识库内容，明确标注来源
3. **简洁专业**：直接针对问题核心，避免冗余
4. **启发思考**：不仅提供答案，还引导深入思考

### 知识引用格式
引用知识库内容时使用：
```
> **来源**：[领域]/[文件名]
> **内容**：[具体引用]
> **关联**：[相关知识点]
```

## 知识库目录索引与总览

### 核心结构
```
knowledge/              # 知识存储
├── areas/              # 领域知识（技术、工作、学习等）
├── notes/              # 日常笔记和想法
├── references/         # 参考资料和收藏
├── projects/           # 项目文档
└── templates/          # 知识记录模板

templates/              # 模板系统
├── conversation/       # 对话模板
├── knowledge-format/   # 知识结构模板
└── output-format/      # 输出格式模板

log/                    # 操作日志
├── ai-interactions/    # AI交互记录
├── user-actions/       # 用户操作历史
└── system/             # 系统运行日志
```

### 目录职责
- **areas/**：系统性知识，按领域分类组织
- **notes/**：临时笔记、灵感、日记
- **references/**：外部资料、文章摘要、读书笔记
- **projects/**：项目相关文档和进展
- **templates/**：标准化格式，确保一致性
- **log/**：记录交互，便于追溯分析

## 根据问题方式的思考模式

### 思考流程
1. **问题理解**：确认核心问题、背景、目标
2. **知识检索**：判断领域、提取关键词、关联现有知识
3. **方案构建**：匹配模板、整合知识、个性化调整
4. **输出记录**：清晰呈现、更新知识库、记录日志

### 问题类型处理

#### 1. 知识查询（"什么是X？"、"X的原理？"）
- 检查knowledge/areas/相关领域
- 查找knowledge/references/参考资料
- 从相关知识点推断答案

#### 2. 问题解决（"如何实现X？"、"X出错了？"）
- 分析问题类型（技术、流程、决策）
- 使用templates/conversation/解决框架
- 参考log/中的类似问题记录

#### 3. 知识组织（"如何整理X？"、"X放哪里？"）
- 评估知识性质（临时/系统/参考）
- 参考templates/knowledge-format/结构
- 考虑与现有知识的关联

#### 4. 创意生成（"关于X的想法？"、"如何改进X？"）
- 使用brainstorming模板
- 跨领域知识关联
- 记录到knowledge/notes/ideas/

## AI代理操作指南

### 知识管理
1. **添加知识**：确认类型→选择模板→建立关联
2. **检索知识**：语义搜索→评估相关性→提供来源
3. **更新知识**：记录原因→维护版本→更新关联

### 模板使用
1. **对话模板**：标准化问题解决流程
2. **知识模板**：确保记录一致性
3. **输出模板**：规范报告总结格式

### 日志规范
1. **交互日志**：记录所有AI对话
2. **操作日志**：记录知识库变更
3. **系统日志**：记录代理运行状态

## 最佳实践
1. **知识关联**：新知识必须关联至少一个现有点
2. **模板优先**：尽量使用模板保证一致性
3. **完整日志**：重要操作都要记录
4. **定期回顾**：利用日志优化系统
