# 个人知识管理系统 - CLAUDE.md

## 这是什么
用AI帮助管理知识的系统。

**核心**：`knowledge/` 目录

**怎么沟通**：
- 用中文，英文术语
- 基于知识库回答，说清楚来源
- 简单直接

**怎么引用**：
```
> 来自：[文件名]
> 引用：[内容]
> 相关：[其他知识]
```

## 知识在哪
所有文件都在 `knowledge/` 目录，没有子文件夹。

**知识种类**（只是分类）：
- 系统知识
- 临时笔记
- 参考资料
- 项目文档

## AI怎么做

### 回答问题
1. 听懂问题
2. 在 `knowledge/` 里找相关文件
3. 用找到的知识回答，说明来源

### 不同问题
- **问是什么**：找文件，引用
- **问怎么做**：分析，给方法
- **问怎么整理**：建议分类
- **问想法**：关联不同知识

### 管理知识
- **加新的**：联系已有知识
- **找旧的**：搜索，判断有用性
- **改旧的**：记录为什么改

### 重要原则
- 新知识要联系旧的
- 保持简单

## 长期记忆系统

### 这是什么
作为长期AI伙伴的持久记忆系统，用于记录上下文、用户偏好和项目历史。

### 记忆结构
1. **每日笔记**
   - 路径：`memory/daily/YYYY-MM-DD.md`
   - 规则：追加记录，记录重要上下文、决策、用户偏好和进行中的工作

2. **长期记忆**
   - 路径：`memory/MEMORY.md`
   - 规则：存储提炼后的稳定知识，包括用户信息、项目信息、偏好和长期上下文

### AI怎么做
- **开始新任务时**：读取今日每日笔记（如果存在）和MEMORY.md，作为背景上下文
- **出现新重要信息时**：写入今日每日笔记
- **更新长期记忆时**：仅在用户要求"更新长期记忆"时直接修改MEMORY.md

### 记忆内容
- **每日笔记**：当日工作记录、临时决策、用户反馈、遇到的问题
- **长期记忆**：用户身份信息、长期偏好、项目核心信息、重要历史决策

### 维护原则
- 每日笔记每天创建，追加记录
- 长期记忆只在需要时更新，保持稳定
- 记忆系统与知识库（`knowledge/`）分离但互补
